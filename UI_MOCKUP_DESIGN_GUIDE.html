UI Mockup Mastery

How to create screenshot-quality UI mockups, dashboards, terminals, and app frames using nothing but HTML and CSS. This guide is written for AI coding agents — copy patterns, adapt tokens, ship beautiful documentation.

🤖
For AI Agents reading this: Every component in this document is self-contained HTML+CSS. All patterns are copy-paste ready. No JavaScript frameworks, no image assets, no external UI libraries required. If in doubt — use dark backgrounds, blur effects, and monospace fonts. The rest flows naturally.
💡
The core trick: Real screenshots look like they do because of border-radius, subtle box-shadow, a top "chrome bar" with colored dots, and a darker-than-content background frame. Replicate that structure and the eye reads it as authentic UI.
🎨 Design Token System

Always start with CSS custom properties. Tokens are the foundation — they make your mockups themeable, consistent, and easy for agents to adapt.

Dark Theme Color Scale

Use a layered background system — 3–4 levels of darkness create natural depth without needing images or gradients everywhere.

--bg (Page) #07070a
--bg-2 (Section) #0d0d12
--bg-3 (Card bg) #111118
--bg-card (Glass) rgba(14,14,20,0.9)

Each accent color has three forms: --blue (full), --blue-dim (10% opacity background), and --blue-bd (border). Always apply them as a trio.

CSS 
:root {
  /* Backgrounds — 4 levels of depth */
  --bg:        #07070a;
  --bg-2:      #0d0d12;
  --bg-3:      #111118;
  --bg-card:   rgba(14, 14, 20, 0.9);

  /* Borders — opacity layers, never solid grey */
  --border:     rgba(255, 255, 255, 0.07);
  --border-med: rgba(255, 255, 255, 0.12);

  /* Each accent = 3 tokens */
  --green:     #10b981;
  --green-dim: rgba(16, 185, 129, 0.10);
  --green-bd:  rgba(16, 185, 129, 0.22);

  /* Text — 4 levels */
  --text:      #f1f1f5;   /* Headings */
  --text-muted:#9ca3af;   /* Body copy */
  --text-dim:  #52525b;   /* Labels, hints */
  --text-xdim: #3f3f46;   /* Disabled states */
}
Aa Typography Rules

Three fonts. One for display, one for body, one for code. Never use browser defaults in design documents.

RoleFontWeightUse Case
DisplaySpace Grotesk500–700Page titles, section headings, logo text, numbers
BodyInter300–700All paragraph text, labels, UI copy, nav items
MonoFira Code400–600Code blocks, terminal output, API URLs, values
HTML — Google Fonts Import 
<link href="https://fonts.googleapis.com/css2?family=Fira+Code:wght@400;600&family=Inter:wght@400;500;600;700;800&family=Space+Grotesk:wght@500;700&display=swap" rel="stylesheet">
💡
Gradient text trick: Apply background: linear-gradient(...); -webkit-background-clip: text; -webkit-text-fill-color: transparent; to any heading for a premium look. Works best on large bold text.

Glass card CSS — the key properties 
/* 1. Gradient background with coloured blobs behind the card */
.glass-bg {
  position: absolute; inset: 0;

.glass-card {

}

</> Syntax Highlighting System

One reusable set of CSS classes for colouring code inline. Apply to <span> tags wrapping tokens. No JavaScript parsers needed.

ClassColourUse for
.t-tagBlue #60a5faHTML tags, selectors
.t-attrViolet #a78bfaHTML attributes, JSON keys
.t-val / .t-strGreen #34d399String values, URLs
.t-propOrange #fb923cCSS properties, numbers
.t-numPink #f472b6Numbers, booleans
.t-kwRose #f43f5eKeywords (const, let, if)
.t-fnLavender #c4b5fdFunction names
.t-cmtDark greyComments — italic
.j-keyVioletJSON object keys
.j-strGreenJSON string values
🤖
Agent instruction: When generating code blocks, wrap each token type in a <span class="t-*">. It's verbose to write but the visual payoff is enormous. For HTML attributes write <span class="t-attr">href</span>=<span class="t-val">"..."</span>.
🌑 Depth, Shadows & Layering

The perception of depth comes from three things working together. Get all three right and any flat div feels tangible.

1. Background Scale

Page → Section → Card → Element. Each level is a few shades lighter. Never jump more than one level per nesting layer.

#07070a base#111118 card
2. Shadow Formula

Use two shadows: a large soft one for elevation, plus an inner white-opacity line for the "highlight edge".

0 24px 60px rgba(0,0,0,.5)
3. Border as Light

Never use solid grey borders. Use rgba(255,255,255,0.07). It reads as a light rim, not a dividing line.

rgba(255,255,255,0.07)
The layered shadow pattern 
/* Floating card — 3-layer shadow */
.card-elevated {
  box-shadow:
    0 2px 8px rgba(0,0,0,0.2),          /* Close shadow */
    0 12px 32px rgba(0,0,0,0.35),        /* Main elevation */
    inset 0 1px 0 rgba(255,255,255,0.06); /* Top highlight */
}

/* Ambient glow — accent colour version */
.glow-orange {
  box-shadow: 0 0 30px rgba(249, 115, 22, 0.25);
}
.glow-violet {
  box-shadow: 0 0 30px rgba(139, 92, 246, 0.3);
}
🤖 Agent Quick-Reference

The distilled rules for any AI agent generating HTML/CSS design documents from scratch.

🤖
When generating a design document, always follow this order:
1 → Define CSS tokens in :root first. 2 → Import Google Fonts. 3 → Build sidebar + scrollbar. 4 → Lay out content sections. 5 → Add components. 6 → Polish with ambient glows and hover states.
RuleWhat to doNever do
BackgroundsUse #07070a as page base, 4 depth levelsUse pure black #000 or plain dark grey
Bordersrgba(255,255,255,0.07) — opacity whiteSolid grey: #333, #444
Text4 levels: #f1f1f5 → #9ca3af → #52525b → #3f3f46Pure white #fff for body copy
FontsSpace Grotesk + Inter + Fira CodeArial, Helvetica, or browser defaults
Frame chromeTraffic light dots: #ff5f57 / #febc2e / #28c840Single-colour or grey dots
ShadowsMulti-layer: close + ambient + inset highlightSingle box-shadow with grey
AccentsUse 3-token system: full / dim / borderUse accent colour at 100% opacity everywhere
Code blocksSpan-wrap each token type with .t-* classesDump plain text into <pre>
Glassmorphismbackdrop-filter:blur() + rgba(255,255,255,0.07) bgSolid opaque backgrounds behind glass
Chartsalign-items:flex-end container + height inline on barsImport Chart.js or any JS charting library
💡
The one-sentence cheat code: Dark layered backgrounds + opacity-white borders + multi-shadow depth + monospace Fira Code + Space Grotesk display font + blur glass panels = any HTML page looks like a $10M SaaS product's documentation.
⚠️
Safari note: Always add -webkit-backdrop-filter alongside backdrop-filter. Safari still requires the prefixed version for glassmorphism to work.
